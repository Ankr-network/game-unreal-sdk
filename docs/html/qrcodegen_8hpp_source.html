36/*
37 * A segment of character/binary/control data in a QR Code symbol.
38 * Instances of this class are immutable.
39 * The mid-level way to create a segment is to take the payload data
40 * and call a static factory function such as QrSegment::makeNumeric().
41 * The low-level way to create a segment is to custom-make the bit buffer
42 * and call the QrSegment() constructor with appropriate values.
43 * This segment class imposes no length restrictions, but QR Codes have restrictions.
44 * Even in the most favorable conditions, a QR Code can only hold 7089 characters of data.
45 * Any segment longer than this is meaningless for the purpose of generating QR Codes.
46 */
47class QrSegment final {
48

53 */

60 public: static const Mode BYTE;
61 public: static const Mode KANJI;
62 public: static const Mode ECI;

67 // The mode indicator bits, which is a uint4 value (range 0 to 15).
68 private: int modeBits;
69
70 // Number of character count bits for three different version ranges.
71 private: int numBitsCharCount[3];

74 /*-- Constructor --*/
75
76 private: Mode(int mode, int cc0, int cc1, int cc2);

221/*
222 * A QR Code symbol, which is a type of two-dimension barcode.
223 * Invented by Denso Wave and described in the ISO/IEC 18004 standard.
224 * Instances of this class represent an immutable square grid of dark and light cells.
225 * The class provides static factory functions to create a QR Code from text or binary data.
226 * The class covers the QR Code Model 2 specification, supporting all versions (sizes)
227 * from 1 to 40, all 4 error correction levels, and 4 character encoding modes.
228 *
229 * Ways to create a QR Code object:
230 * - High level: Take the payload data and call QrCode::encodeText() or QrCode::encodeBinary().
231 * - Mid level: Custom-make the list of segments and call QrCode::encodeSegments().
232 * - Low level: Custom-make the array of data codeword bytes (including
233 * segment headers and final padding, excluding error correction codewords),
234 * supply the appropriate version number, and call the QrCode() constructor.
235 * (Note that all ways require supplying the desired error correction level.)
236 */
237class QrCode final {

511/*
512 * Thrown when the supplied data does not fit any QR Code version. Ways to handle this exception include:
513 * - Decrease the error correction level if it was greater than Ecc::LOW.
514 * - If the encodeSegments() function was called with a maxVersion argument, then increase
515 * it if it was less than QrCode::MAX_VERSION. (This advice does not apply to the other
516 * factory functions because they search all versions up to QrCode::MAX_VERSION.)
517 * - Split the text data into better or optimal segments in order to reduce the number of bits required.
518 * - Change the text or binary data to be shorter.
519 * - Change the text to fit the character set of a particular segment mode (e.g. alphanumeric).
520 * - Propagate the error upward to the caller/user.
521 */
